객체지향 프로그래밍에 대해서 공부
-----------------------------
## 1.개요: 
1-1.객체 지향 프로그래밍: 객체 지향 프로그래밍(OOP)은 프로그램 데이터와 그 데이터를 조작하는 함수를 하나의 객체"(Object)"로 묶어, 객체들 간의 상호작용으로 로직을 구성하는 패러다임이다. 
*로직:상호작용의 결과를 정의하는 규칙
*패러다임:이론적 틀,공유된 모범 사례를 뜻함
*****
## 2.특징
2-1.추상화: 불필요한 사항들을 제거하고 공통성과 본진을 모아 추출하여 표헌한다.
2-2.상속: 기존의 클래스를 재활용하여 새로운 클래스를 작성한다.
2-3.다형성: 한 객체가 여러가지 속성이나 기능이 상황에 따라 여러 가지 형태를 가질 수 있는 성질.
2-4.캡슐화: 클래스 안에 서로 연관있는 속성과 기능들을 하나의 캡슐로 만들어 데이터를 외부로부터 보호하는 것을 뜻 한다.
*****
3.장단점
3-1.장점: 프로그램을 보다 유연하고 변경이 쉽게 만들 수 있음,객체지향 프로그래밍을 통해 만든 프로그램들은 각각의 객체들이 각자의 독립적인 역할을 가지기 때문에 코드의 변경을 최소화하고 유지보수를 하는 데 유리함
3-2.단점: 캡슐화,다향성 등으로 인해 처리 속도가 상대적으로 느릴 수 있음,모든 것을 객체로 모델링 하기 때문에 메모리 사용량이 높음,높은 수준의 추상화와 설걔룰 요구하므로 초기 개발 속도가 느릴 수 있으며,사전 설계가 필수적임.
*****
